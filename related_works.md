# Related Works

本文工作关注的是在外部稳定前端提供位姿的条件下，如何构建一个更适合增量式场景重建的高斯地图后端。因此，相关工作主要从三个方向展开：照片级辐射场重建与高斯表示、增量式视觉高斯重建系统，以及多传感器高斯重建与 Gaussian-LIC 系列方法。

## 1. 照片级辐射场重建与高斯表示

NeRF 及其后续工作证明了辐射场表示在新视角合成和连续外观建模中的优势，但基于体渲染的采样与积分过程代价较高，难以直接满足实时增量式重建的需求。3D Gaussian Splatting（3DGS）以显式高斯原语和可微光栅化替代体积分，在渲染速度和优化效率上取得了明显优势，因此很快成为在线照片级建图的重要基础表示。

围绕 3DGS，已有研究从两个方向持续推进。一类工作关注几何质量，例如 2D Gaussian Splatting（2DGS）、GaussianSurfels 和 PGSR 等方法通过表面化高斯、深度栅格化和法线约束增强几何一致性。其中，2DGS 将体高斯压缩为嵌入三维空间的二维有向平面高斯盘，并通过 ray-splat intersection、深度畸变约束和法线一致性约束显著改善了表面贴合能力。这说明，高斯表示的瓶颈不仅在于渲染效率，也在于几何一致性，而这一点对后续增量式系统尤为关键。另一类工作则关注前馈式高斯先验，例如 pixelSplat、MVSplat、MVSGaussian 和 PixelGaussian 等。这些方法通过概率采样、cost volume 或自适应高斯分配机制，从少量输入图像直接预测可泛化的高斯表示。尽管它们主要面向离线或一次性多视图重建，但其输出所体现的几何与外观先验，为在线系统中的初始化和补全提供了新的可能。

总体而言，这些工作奠定了两个关键事实：其一，高斯表示比传统隐式辐射场更适合作为实时场景重建的后端基础；其二，若要进一步提升在线系统的几何质量，仅依赖原始 3DGS 仍然不够，需要引入更强的表面化表示和可学习先验。

## 2. 增量式视觉高斯重建系统

在增量式视觉系统中，Gaussian Splatting SLAM（亦称 MonoGS）首次证明了 3DGS 可以作为单目在线系统的统一地图表示，将跟踪、建图和渲染耦合在同一高斯地图上。随后，GS-SLAM 和 SplaTAM 分别在 RGB-D 场景下进一步展示了 3DGS 在高效地图扩展、结构化更新与实时渲染中的优势，说明高斯地图不仅适合做静态重建，也适合持续吸收新观测。

在单目和全局一致性方面，Splat-SLAM 强调全局位姿和深度更新对高斯地图的反馈作用，SplatMAP 则提出基于 SLAM 动态更新的自适应致密化策略，并通过几何引导损失提升局部结构质量。与这些方法相比，DROID-Splat 更进一步地体现出前端和地图后端的模块化解耦：强跟踪模块负责提供稳定姿态，而高斯地图后端专注于渲染和增量优化。这一思路与本文的系统设定较为接近，即不把里程计本身作为主要创新点，而把重点放在地图表示与地图更新机制上。

近年来，更多工作开始围绕“怎样让地图更可靠”引入额外先验。GI-SLAM 通过 IMU loss 和关键帧选择策略增强增量式地图质量；FeatureSLAM 将基础模型特征嵌入高斯地图，使地图同时承载外观和高层语义；WildGS-SLAM 在动态环境中通过不确定性建模抑制动态区域对地图的污染；SEGS-SLAM 则从结构初始化与外观嵌入两个角度提升高斯地图的稳定性和表达能力。这些工作共同说明，增量式高斯系统的核心竞争点已经从“能否在线”转向“地图初始化是否可靠、地图更新是否可控、地图几何是否足够可用”。

然而，上述方法大多仍建立在 3DGS 体高斯表示之上。尽管它们通过深度、结构、惯性、语义和不确定性等先验提升了地图质量，但尚未从表示层根本解决体高斯带来的几何不一致问题。这也为将 2DGS 引入增量式系统留下了明确空间。

## 3. 多传感器高斯重建与 Gaussian-LIC 系列

对于户外和大尺度场景，单纯依赖 RGB 或 RGB-D 难以同时满足覆盖范围、几何精度与在线稳定性要求，因此 LiDAR、IMU 与相机融合逐渐成为高斯重建的重要路线。Gaussian-LIC 是该方向的代表性起点，它将 LiDAR-Inertial-Camera 融合与 3DGS 建图结合起来，并通过 LiDAR 点与在线三角化视觉点共同初始化高斯，以缓解 LiDAR 视野之外的欠重建问题。该方法的重要意义在于，它将多传感器系统真正转化为“支撑高斯地图持续增长的观测来源”，而不只是将高斯地图当作离线可视化结果。

在此基础上，GS-LIVM 和 GS-LIVO 分别从大规模映射和实时资源约束出发，进一步发展了多传感器高斯地图。GS-LIVM 利用 Gaussian Process Regression 和协方差中心化设计处理稀疏且分布不均的 LiDAR 观测，并将协方差用于高斯尺度与旋转初始化；GS-LIVO 则构建分层体素化全局高斯图与局部滑动窗口高斯集合，仅优化窗口内地图，以降低显存和计算开销。这些工作表明，多传感器高斯系统的关键不只是“有没有 LiDAR”，而在于如何利用多模态观测支撑高斯的初始化、扩展与局部优化。

Gaussian-LIC2 则进一步将研究重点从单纯的视觉渲染质量推进到视觉质量、几何准确性和实时性能的联合优化。该方法明确指出，多数现有 LiDAR-Inertial-Camera 高斯系统仍高度依赖稠密 LiDAR、相机与 LiDAR 视场对齐，以及仅由 LiDAR 点驱动的高斯初始化，因此在 LiDAR 盲区、稀疏激光场景和几何渲染质量上仍存在明显不足。为此，Gaussian-LIC2 引入轻量级零样本深度模型，将 RGB 外观与稀疏 LiDAR 融合为稠密深度，用于高斯初始化。这一设计已经体现出“前馈先验服务于增量式高斯建图”的思路，但其底层地图表示仍然是 3DGS。

从本文关注的方向来看，多传感器 Gaussian-LIC 系列已经解决了一个非常关键的问题，即如何在外部稳定前端支撑下持续完成照片级增量建图；但它们仍未充分解决两个问题：其一，3DGS 在表面建模上的几何不一致会限制深度、法线和后续几何任务的质量；其二，现有学习型先验大多只被用作深度补全，而尚未系统地扩展到高斯的尺度、方向和扩展区域筛选。

## 4. 本文工作的定位

不同于将主要贡献放在前端里程计或统一 SLAM 框架上的工作，本文工作的重点是：在外部稳定前端提供位姿的前提下，围绕增量式场景重建后端改进高斯地图表示、初始化与扩展机制。具体而言，本文与现有工作的区别主要体现在两点。

第一，本文不再沿用 3DGS 体高斯作为默认地图原语，而是进一步引入 2DGS 的表面化表示思想，以改善增量式系统中长期存在的几何不稳定、深度漂移和表面模糊问题。第二，本文不把前馈高斯方法视为替代在线系统的独立重建器，而是将其重新定位为 initialize/extend 阶段的先验模块，用于弥补 LiDAR 盲区、低纹理区域和早期低视差阶段的初始化不足。换言之，本文的核心目标并不是重新设计一个更复杂的 SLAM 前端，而是在 Gaussian-LIC 一类多传感器增量建图框架内，通过 2DGS 与前馈先验的结合，构建一个几何更一致、初始化更充分、在线更新更高效的高斯地图后端。
